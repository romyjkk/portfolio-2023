INTRODUCTION

The most challenging project I did at my internship. I had to use the Fetch API and learn PHP to create a sign up form with form validation (not the standard HTML options) and send an email with the form data attached. I was allowed to give it my own twist, which I did!

PROCESS

Because there was so much that was new to me, I decided to start by looking into PHP and the Fetch API, so I at least understood what it was all about. I also thought about the form itself, what inputs it would have and how I would validate it.

I got some help with starting it all up, so the Fetch API would send the data to my PHP form validation file, where I could work out the form validation and also add the code to send a mail. The PHP form validation went pretty good, there were only a few things I needed help with.

When the form validation was done, I had to find a way to show the feedback messages on the page. This was eventually done by creating an if-statement, which says that if there’s an error, it will loop through the different fields and if there’s an error, it will create a div containing the error message. This was one of the most challenging parts for me, and I got some help with it.

Throughout the project I styled the form, added some fun animations and added an easter egg.

RESULTS

With no prior knowledge of PHP, I made a sign up form with working PHP form validation and errors. When the sign up is complete, an email is sent containing the data used in the form.
